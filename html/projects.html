SubOps is one of my recent projects. I wanted to learn how to wrie subtitles and help out with translating spiritually important videos to my native language. Writing subtitles is not a so easy task as one might think. Besides content, it involves timing for each sibtitle line. I ended up using the Audacity software to view audio waves in a particular video. The problem was that the program has so called labels which do give start and end time but formatted differently. Because of that I wrote one of the features that would convert label time format to SRT.

Once upon a time, at one Seventh-Day-Adventist Church, I was tasked to manage a mutual fund. One of the duties was to write out receipts to a person who was enrolling or contributing to the fund. At that time, the fund consisted of many people and writing out receipts by hand would take lots of time. At one point I decided to try to write a simple program that would both, give me more practice in writing computer programs and help me save some time preparing receipts. I found a pdf library for the Rust programming language, studied it, and with strength and wisdom from the Holy Spirit, was able to accomplish my goal.

The riscv continuous integration project was a Cap Stone project. It involved a team of 8 people who worked together to automate the process of testing a picolibc library for riscv instruction set used in embedded devices. My contribution was a Docker file that helped us build a portable environment to run the tests in. This was my first serious dive into a containerized solution and I was glad my God helped me contribute to the project.
